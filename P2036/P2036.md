# P2036 [COCI 2008/2009 #2] PERKET 题解

## 题目描述

Perket 是一种流行的美食。为了做好 Perket，厨师必须谨慎选择食材，以在保持传统风味的同时尽可能获得最全面的味道。你有 $n$ 种可支配的配料。对于每一种配料，我们知道它们各自的酸度 $s$ 和苦度 $b$。当我们添加配料时，总的酸度为每一种配料的酸度总乘积；总的苦度为每一种配料的苦度的总和。

众所周知，美食应该做到口感适中，所以我们希望选取配料，以使得酸度和苦度的绝对差最小。

另外，我们必须添加至少一种配料，因为没有任何食物是只以水为配料的。

### 输入格式

第一行一个整数 $n$，表示可供选用的食材种类数。

接下来 $n$ 行，每行 $2$ 个整数 $s_i$ 和 $b_i$，表示第 $i$ 种食材的酸度和苦度。

### 输出格式

一行一个整数，表示可能的总酸度和总苦度的最小绝对差。

### 输入输出样例

#### 输入 #1

text

```
1
3 10
```

#### 输出 #1

text

```
7
```

#### 输入 #2

text

```
2
3 8
5 8
```

#### 输出 #2

text

```
1
```

#### 输入 #3

text

```
4
1 7
2 6
3 8
4 9
```

#### 输出 #3

text

```
1
```

### 说明/提示

对于第三组样例，选择最后三种食材，此时的总酸度为 $2 \times 3 \times 4 = 24$，总苦度为 $6+8+9=23$，差值为 $1$。

#### 数据规模与约定

对于 $100%$ 的数据，有 $1 \leq n \leq 10$，且将所有可用食材全部使用产生的总酸度和总苦度小于 $1 \times 10^9$。

## AC代码

cpp

```
#include <algorithm>
#include <cstdio>
#include <cmath>
using namespace std;

int a[15], b[15], n, ans = 1 << 30; // a表示酸度，b表示苦度，ans为最终答案

void dfs(int i, int sj, int th) {
    if (i > n) {
        if (!th) // 如果没有选任何一种材料
            return;
        ans = min(ans, abs(sj - th)); // 否则更新答案
        return;
    }
    
    // 两个分支：不选 和 选
    dfs(i + 1, sj, th);           // 不选当前配料
    dfs(i + 1, sj * a[i], th + b[i]); // 选当前配料
}

int main() {
    scanf("%d", &n);
    for (int i = 1; i <= n; i++)
        scanf("%d %d", &a[i], &b[i]);
    
    dfs(1, 1, 0); // 搜索，酸度初始值为1（乘法单位元），苦度初始值为0
    
    printf("%d", ans);
    return 0;
}
```

------

## 代码思路

1. **状态设计**：`dfs(i, sj, th)` 表示处理到第 i 种配料，当前总酸度为 sj，总苦度为 th
2. **递归终止**：当 i > n 时，表示所有配料都处理完毕
3. **合法性检查**：如果 th == 0，说明没有选择任何配料，直接返回
4. **答案更新**：否则计算 |sj - th|，更新最小值
5. **两个分支**：
   - **不选**当前配料：sj 和 th 保持不变，处理下一种配料
   - **选**当前配料：sj 乘以当前酸度，th 加上当前苦度，处理下一种配料

这种 DFS 的本质是**枚举所有子集**，每个配料都有"选"和"不选"两种状态。

## 我的错误点

我最初的错误代码：

cpp

```
void dfs(int index, int total_s, int total_b, int cnt, struct peiliao P[]) {
    if (cnt > 0) {
        ans = min(ans, abs(total_s - total_b));
    }
    for (int i = index; i < n; i++) {        
        if (cnt == 0) {
            dfs(i + 1, P[i].s, P[i].b, 1, P);
        } else {
            dfs(i + 1, total_s * P[i].s, total_b + P[i].b, cnt + 1, P);
        }
    }
}
```

### 主要错误：

1. **❌ 缺少"不选"的分支**：代码只考虑了"选"的情况，没有对应的"不选"分支
2. **❌ 搜索不完整**：只能生成从起始点开始的连续组合，无法生成所有可能的子集
3. **❌ 递归终止条件不明确**：虽然通过 for 循环会自然结束，但逻辑不够清晰
4. **❌ 曾使用 i++ 导致无限递归**：在递归参数中使用后置递增运算符造成 MLE

## DFS 的思路详解

### 1. 为什么用 DFS？

n ≤ 10，数据范围很小，可以用 DFS 枚举所有可能的配料组合（2^n 种情况）。

### 2. 状态定义

text

```
dfs(当前处理到的配料索引, 当前总酸度, 当前总苦度)
```

### 3. 决策树

对于每种配料，有两个选择：

text

```
                    开始 (i=1, sj=1, th=0)
                    /              \
             不选配料1              选配料1
              /      \              /      \
        不选配料2  选配料2     不选配料2  选配料2
        /    \     /    \      /    \     /    \
      ...   ...  ...   ...   ...   ...  ...   ...
```

### 4. 递归终止条件

当处理完所有配料（i > n）时，检查是否至少选了一种配料（th > 0），如果是则更新答案。

### 5. 关键点

- **酸度初始值为 1**：因为酸度是乘积，1 是乘法单位元
- **苦度初始值为 0**：因为苦度是求和，0 是加法单位元
- **必须至少选一种配料**：通过判断 th > 0 来保证

## 经验总结

### 1. DFS 的基本模板

cpp

```
void dfs(当前状态) {
    if (终止条件) {
        更新答案;
        return;
    }
    
    // 枚举所有可能的选择
    for (所有可能的决策) {
        更新状态;
        dfs(新状态);
        恢复状态; // 如果需要回溯
    }
}
```

### 2. 本题的 DFS 要点

- 每个配料只有两种状态，不需要显式回溯（参数传递的是新值）
- 必须包含"选"和"不选"两个分支，才能枚举所有子集
- 递归终止条件要放在函数开头

### 3. 常见错误避免

| 错误类型         | 正确做法                     |
| ---------------- | ---------------------------- |
| 缺少"不选"分支   | 每个决策点都要考虑所有可能性 |
| 递归参数使用 i++ | 直接用 i+1 传递新值          |
| 忘记至少选一种   | 用 cnt 或 th 标记是否选过    |
| 酸度初始值设 0   | 酸度是乘积，应初始为 1       |

### 4. 小技巧

- 用 `1 << 30` 表示无穷大，比 1000000 更规范
- 酸度乘积可能很大，但题目保证在 int 范围内
- 可以只用酸度和苦度两个参数，不需要额外的 cnt（通过 th 是否为 0 判断）

### 5. 思考方式

当遇到"选择若干元素"的问题时：

1. 确定每个元素有几种选择（通常是选/不选）
2. 设计状态表示当前进度和结果
3. 确定递归终止条件
4. 编写递归函数，枚举所有选择

DFS 的本质就是**暴力枚举**，关键是要保证**不重不漏**地枚举所有可能的情况。
