# P1094 [NOIP 2007 普及组] 纪念品分组

## 题目描述

元旦快到了，校学生会让乐乐负责新年晚会的纪念品发放工作。为使得参加晚会的同学所获得的纪念品价值相对均衡，他要把购来的纪念品根据价格进行分组，但每组最多只能包括两件纪念品，并且每组纪念品的价格之和不能超过一个给定的整数。为了保证在尽量短的时间内发完所有纪念品，乐乐希望分组的数目最少。

你的任务是写一个程序，找出所有分组方案中分组数最少的一种，输出最少的分组数目。

## 输入格式

共 $n+2$ 行：

第一行包括一个整数 $w$，为每组纪念品价格之和的上限。

第二行为一个整数 $n$，表示购来的纪念品的总件数 $G$。

第 $3\sim n+2$ 行每行包含一个正整数 $P_i$ 表示所对应纪念品的价格。

## 输出格式

一个整数，即最少的分组数目。

## 输入输出样例

**输入 #1**

text

```
100
9
90
20
20
30
50
60
70
80
90
```

**输出 #1**

text

```
6
```

## 数据范围

- $1\le n\le 3\times 10^4$
- $80\le w\le 200$
- $5\le P_i \le w$

## 思路分析

### 为什么能想到双指针？

这道题有两个关键特征：

1. **每组最多两件物品** - 这大大简化了问题，每个组要么是单件，要么是两件
2. **要最小化组数** - 直觉上，我们应该尽可能让两件物品配对，因为两件一组比两件单独成组能减少组数

当面对这种"配对"问题，特别是涉及大小关系时，**排序 + 双指针**是一个非常自然的思路：

- 排序让数据有序，便于分析大小关系
- 双指针从两端向中间移动，可以高效地尝试"最小配最大"的组合

### 贪心算法的体现

本题是典型的贪心算法应用：

**贪心策略**：每次尝试将当前最贵的物品与当前最便宜的物品配对

**为什么贪心是正确的？**

1. 如果最贵的物品可以和最便宜的物品配对（即价格之和 ≤ w），那么将它们放在一组是最优的。因为：
   - 最便宜的物品与其他任何物品配对的可能性更大
   - 让最贵的物品与最便宜的物品配对，可以为中间价格的物品留下更多配对机会
2. 如果最贵的物品连最便宜的物品都配不上，那么它只能单独成组，因为没有其他物品能跟它配对了（其他物品价格都 ≥ 最便宜的价格）

这种"最贵配最便宜"的策略保证了分组数最小。

## AC代码（双指针法）

cpp

```
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int w, n;
    cin >> w >> n;
    
    vector<int> prices(n);
    for (int i = 0; i < n; i++) {
        cin >> prices[i];
    }
    
    sort(prices.begin(), prices.end());
    
    int left = 0, right = n - 1;
    int groups = 0;
    
    while (left <= right) {
        if (prices[left] + prices[right] <= w) {
            // 可以配对
            left++;
            right--;
        } else {
            // 最贵的单独一组
            right--;
        }
        groups++;
    }
    
    cout << groups << endl;
    
    return 0;
}
```

**复杂度分析**：

- 时间复杂度：$O(n \log n)$，主要来自排序
- 空间复杂度：$O(1)$，只用了常数个变量

## 对比分析：双重循环代码

这是你写的双重循环代码：

cpp

```
#include <bits/stdc++.h>
using namespace std;
int w,n;

int main()
{
	cin.tie(0);
	
	cin>>w;
	cin>>n;
	vector<int> visi(n,0);
	vector<int> jiage(n);
	for(int i=0;i<n;i++)cin>>jiage[i];
	sort(jiage.begin(),jiage.end());
	int ans=0;
	for(int i=0;i<n;i++)
		{
			for(int j=n-1;j>=0;j--)
				{
					if(visi[j]==0){
						if(jiage[i]+jiage[j]<=w){
						visi[i]=1;
						visi[j]=1;
						ans++;
						break;
						}	
						else{
							visi[j]=1;
							ans++;
						}
					}
					
				}
		}
	cout<<ans;
	return 0;
}
```

### 双重循环的问题

1. **时间复杂度高**：最坏情况下 $O(n^2)$，当 $n=3\times 10^4$ 时，可能需要执行约 9 亿次操作，在时间限制内可能超时
2. **逻辑复杂**：需要维护 `visi` 数组来标记已使用的物品
3. **不必要的循环**：内层循环每次都从末尾开始扫描，做了很多重复工作
4. **配对策略不够优化**：虽然思路正确，但实现效率低

### 两种方法的对比

| 对比项     | 双重循环法   | 双指针法      |
| ---------- | ------------ | ------------- |
| 时间复杂度 | $O(n^2)$     | $O(n \log n)$ |
| 空间复杂度 | $O(n)$       | $O(1)$        |
| 代码简洁度 | 较复杂       | 简洁明了      |
| 思路清晰度 | 需要标记数组 | 直观易懂      |
| 数据适应性 | 仅适合小数据 | 适合大数据    |

## 总结

这道题的精华在于：

1. **排序预处理**，让数据有序
2. **双指针技巧**，高效地从两端向中间扫描
3. **贪心思想**，每次尝试最优的配对方案

当你遇到"分组"、"配对"类问题，特别是限制每组最多两个物品时，双指针 + 贪心是一个非常强大的工具组合。它不仅能保证正确性，还能提供极佳的时间效率。
