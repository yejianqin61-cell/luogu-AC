# P1208 [USACO1.3] 混合牛奶 Mixing Milk

## 题目描述

由于乳制品产业利润很低，所以降低原材料（牛奶）价格就变得十分重要。帮助 Marry 乳业找到最优的牛奶采购方案。

Marry 乳业从一些奶农手中采购牛奶，并且每一位奶农为乳制品加工企业提供的价格可能相同。此外，就像每头奶牛每天只能挤出固定数量的奶，每位奶农每天能提供的牛奶数量是一定的。每天 Marry 乳业可以从奶农手中采购到小于或者等于奶农最大产量的整数数量的牛奶。

给出 Marry 乳业每天对牛奶的需求量，还有每位奶农提供的牛奶单价和产量。计算采购足够数量的牛奶所需的最小花费。

注：每天所有奶农的总产量不少于 Marry 乳业的需求量。

## 输入格式

第一行二个整数 $n,m$，表示需要牛奶的总量，和提供牛奶的农民个数。

接下来 $m$ 行，每行两个整数 $p_i,a_i$，表示第 $i$ 个农民牛奶的单价，和农民 $i$ 一天最多能卖出的牛奶量。

## 输出格式

单独的一行包含单独的一个整数，表示 Marry 的牛奶制造公司拿到所需的牛奶所要的最小费用。

## 输入输出样例

**输入 #1**

text

```
100 5
5 20
9 40
3 10
8 80
6 30
```

**输出 #1**

text

```
630
```

## 数据范围

对于 $100%$ 的数据：

- $0 \le n,a_i \le 2 \times 10^6$
- $0\le m \le 5000$
- $0 \le p_i \le 1000$

## AC代码

cpp

```
#include <bits/stdc++.h>
using namespace std;

struct nai {
    int danjia;   // 单价
    int shuliang; // 数量
};

// 按单价升序排序
bool compare(const nai& a, const nai& b) {
    return a.danjia < b.danjia;
}

int main() {
    cin.tie(0);
    int n, m;
    cin >> n >> m;
    
    vector<nai> niunai(m);
    for (int i = 0; i < m; i++) {
        cin >> niunai[i].danjia >> niunai[i].shuliang;
    }
    
    // 贪心策略：优先选择单价低的牛奶
    sort(niunai.begin(), niunai.end(), compare);
    
    int fee = 0;      // 总花费
    int total = 0;    // 已采购的牛奶总量
    
    for (int i = 0; i < m; i++) {
        total += niunai[i].shuliang;
        
        if (total == n) {
            // 刚好达到需求量
            fee += niunai[i].danjia * niunai[i].shuliang;
            break;
        }
        else if (total > n) {
            // 超过了需求量，只需采购部分
            int temp = total - n;  // 多出的数量
            fee += niunai[i].danjia * niunai[i].shuliang - niunai[i].danjia * temp;
            break;
        }
        
        // 还未达到需求量，全部采购
        fee += niunai[i].danjia * niunai[i].shuliang;
    }
    
    cout << fee;
    return 0;
}
```

------

## 思路

这道题是一个经典的**贪心算法**问题。目标是在满足牛奶需求量的前提下，使总花费最小。

**核心思路**：

- 既然要最小化总花费，自然应该优先选择**单价最低**的牛奶
- 将奶农按照牛奶单价从小到大排序
- 从单价最低的奶农开始采购，直到满足需求量为止

**为什么贪心是正确的？**
因为牛奶本身是同质的，没有质量差异，唯一影响选择的就是价格。在总需求量固定的情况下，优先采购便宜的牛奶一定可以得到最优解。这符合贪心算法的"最优子结构"性质。

------

## 贪心实现

**实现步骤**：

1. 定义结构体存储每个奶农的牛奶单价和数量
2. 读取输入数据
3. 使用 `sort` 函数按单价升序排序
4. 遍历排序后的数组：
   - 累加当前奶农的牛奶数量到总量
   - 如果总量等于需求量，加上当前奶农的全部金额后结束
   - 如果总量超过需求量，说明只需要当前奶农的部分牛奶，计算应支付的金额后结束
   - 如果总量小于需求量，加上当前奶农的全部金额，继续循环
5. 输出总花费

**关键代码解析**：


当总量超过需求时，需要计算实际应支付的金额：

- `niunai[i].danjia * niunai[i].shuliang` 是当前奶农牛奶的全价
- `niunai[i].danjia * temp` 是多余牛奶的金额（不需要支付）
- 两者相减即为实际应支付的金额

## 经验总结

1. **贪心策略的选择**：当问题具有"最优子结构"且局部最优能导致全局最优时，可以考虑使用贪心算法。
2. **结构体的使用**：将相关的数据（单价和数量）封装在结构体中，使代码更加清晰。
3. **排序的重要性**：`sort` 函数配合自定义比较函数，可以方便地实现各种排序需求。
4. **边界条件处理**：
   - 当总量恰好等于需求量时，直接全部采购
   - 当总量超过需求量时，只需要采购部分
   - 数据范围较大时（$2 \times 10^6$），注意使用合适的变量类型（int 足够）
5. **代码优化**：使用 `cin.tie(0)` 可以提高输入输出效率，对于大数据量很有帮助。
6. **逻辑清晰**：循环中的三个分支（等于、大于、小于）覆盖了所有可能的情况，确保算法的正确性。
