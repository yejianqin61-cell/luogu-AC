

# P4995 跳跳！题解

## 题目描述

你是一只小跳蛙，你特别擅长在各种地方跳来跳去。

这一天，你和朋友小F一起出去玩耍的时候，遇到了一堆高矮不同的石头，其中第i块石头的高度为hi，地面的高度是h0=0。你估计着，从第i块石头跳到第j块石头上耗费的体力值为(hi - hj)^2，从地面跳到第i块石头耗费的体力值是(hi)^2。

为了给小F展现你超级跳的本领，你决定跳到每个石头上各一次，并最终停在任意一块石头上，并且小跳蛙想耗费尽可能多的体力值。

当然，你只是一只小跳蛙，你只会跳，不知道怎么跳才能让本领更充分地展现。

不过你有救啦！小F给你递来了一个写着AK的电脑，你可以使用计算机程序帮你解决这个问题，万能的计算机会告诉你怎么跳。

那就请你——会写代码的小跳蛙——写下这个程序，为你NOIp AK踏出坚实的一步吧！

## 输入格式

输入一行一个正整数n，表示石头个数。

输入第二行n个正整数，表示第i块石头的高度hi。

## 输出格式

输出一行一个正整数，表示你可以耗费的体力值的最大值。

## 输入输出样例

样例1：
输入：
2
2 1
输出：
5

样例2：
输入：
3
6 3 5
输出：
49

## 数据范围

对于1 ≤ i ≤ n，有0 < hi ≤ 10^4，且保证hi互不相同。

对于10%的数据，n ≤ 3；
对于20%的数据，n ≤ 10；
对于50%的数据，n ≤ 20；
对于80%的数据，n ≤ 30；
对于100%的数据，n ≤ 300。

## AC代码

cpp

```
#include <bits/stdc++.h>
using namespace std;

int main() {
    cin.tie(0);
    ios::sync_with_stdio(false);
    
    long long n;
    cin >> n;
    
    vector<long long> high(n + 1);
    for (long long i = 1; i <= n; i++) {
        cin >> high[i];
    }
    
    high[0] = 0;  // 地面高度
    sort(high.begin(), high.end());  // 从小到大排序
    
    long long ti = 0;  // 总体力值
    long long i = 0, j = n;  // 双指针：i指向最小，j指向最大
    
    while (i <= j) {
        // 从矮（i）跳到高（j）
        ti += (high[i] - high[j]) * (high[i] - high[j]);
        i++;
        
        if (i > j) break;
        
        // 从高（j）跳到次矮（i）
        ti += (high[i] - high[j]) * (high[i] - high[j]);
        j--;
    }
    
    cout << ti << endl;
    return 0;
}
```

## 解题思路

### 核心思想：贪心算法

这道题的关键在于，要想耗费的总体力值最大，每次跳跃都应该选择与当前石头高度差最大的石头。因为体力值是高度差的平方，差越大，体力值越大。

### 贪心策略的证明

假设当前站在某块石头上，剩下的石头中，最高和最低的石头与当前石头的高度差一定是最大的两个选择。如果当前在最低点，就应该跳向最高点；如果当前在最高点，就应该跳向剩余石头中的最低点。这样交替进行，就能保证每一步都是最优选择。

### 具体实现方法

1. 将地面（高度0）和所有石头的高度放在一起，从小到大排序。
2. 使用两个指针i和j，i指向最矮的石头（包括地面），j指向最高的石头。
3. 按照"最矮→最高→次矮→次高→..."的顺序交替跳跃。
4. 每跳一次，移动对应的指针，直到所有石头都跳到。

### 模拟样例

以样例2为例：n=3，石头高度为[6,3,5]

加入地面后排序：[0,3,5,6]

跳跃过程：

- 从地面(0)跳到最高(6)：体力 = (0-6)^2 = 36
- 从最高(6)跳到次矮(3)：体力 = (6-3)^2 = 9
- 从次矮(3)跳到次高(5)：体力 = (3-5)^2 = 4
- 从次高(5)跳到最后一个(5)：体力 = (5-5)^2 = 0

总体力 = 36 + 9 + 4 + 0 = 49

## 注意事项

### 1. 必须使用long long类型

这是本题最容易出错的地方！很多同学因为没用long long导致WA。

为什么要用long long？

- 石头高度最大为10^4
- 单次跳跃体力最大为(10^4 - 0)^2 = 10^8
- 最多跳跃n+1次，n最大为300
- 总体力最大约为300 × 10^8 = 3 × 10^10

而int类型的最大值只有约2.147 × 10^9，远远不够！所以必须用long long（64位整数）来存储结果。

### 2. 别忘了加入地面高度

地面高度0要加入数组一起排序，因为第一次跳跃是从地面开始的。

### 3. 注意边界条件

当n为奇数时，最后两个指针会指向同一块石头，此时会多一次差为0的跳跃，不影响结果。代码中的if判断可以防止数组越界。

## 时间复杂度

排序的时间复杂度为O(nlogn)，双指针遍历的时间复杂度为O(n)。整体时间复杂度O(nlogn)，完全满足n≤300的数据范围。

## 总结

本题是一个典型的贪心算法问题，通过排序后双指针交替跳跃即可得到最优解。最关键的一点是注意数据范围，一定要使用long long类型存储结果，避免溢出导致的答案错误。

记住：在算法竞赛中，看到较大的数据范围或者涉及平方、累加的操作，第一时间考虑使用long long，这是保证代码正确性的重要习惯。
