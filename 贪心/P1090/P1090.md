## 一、题目分析

**题目要求**：将n堆果子合并成一堆，每次合并两堆，消耗体力为两堆重量之和，求最小总消耗。

**关键点**：

- 每次合并只能选两堆
- 合并后产生的新堆会参与后续合并
- 目标是使总消耗最小

**本质**：这是经典的**哈夫曼树**（最优二叉树）构造问题。

------

## 二、贪心算法思路

### 贪心策略

**每次选择当前重量最小的两堆进行合并**

### 为什么正确？

1. **最优子结构**：如果某次合并不选最小的两堆，那么这次合并的代价会比选最小两堆大，而且较大的堆会在后续被更早地合并，导致总代价更大
2. **贪心选择性质**：局部最优（每次选最小的两堆）能导致全局最优

### 证明思路（反证法）

假设某次合并选了不是最小的两堆(x,y)，而最小的两堆是(a,b)且a≤b≤x≤y：

- 合并(x,y)的代价 = x+y
- 合并(a,b)的代价 = a+b
- 显然 x+y ≥ a+b
- 而且更大的x和y会在后续合并中导致更大代价

## 三、优先队列实现（推荐）

cpp

```
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    // 小根堆：每次自动弹出最小的元素
    priority_queue<long long, vector<long long>, greater<long long>> pq;
    
    for (int i = 0; i < n; i++) {
        long long x;
        cin >> x;
        pq.push(x);
    }
    
    long long ans = 0;
    
    // 合并 n-1 次
    while (pq.size() > 1) {
        // 取出最小的两个
        long long a = pq.top(); pq.pop();
        long long b = pq.top(); pq.pop();
        
        long long sum = a + b;
        ans += sum;        // 累加消耗的体力
        pq.push(sum);      // 新堆放回队列
    }
    
    cout << ans << endl;
    
    return 0;
}
```

### 代码解析

- `priority_queue<long long, vector<long long>, greater<long long>>`：小根堆
- `top()`：取最小元素 O(1)
- `pop()`：弹出最小元素 O(log n)
- `push()`：插入元素 O(log n)

## 四、multiset实现

cpp

```
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    // multiset 自动排序（从小到大），允许重复元素
    multiset<long long> s;
    
    for (int i = 0; i < n; i++) {
        long long x;
        cin >> x;
        s.insert(x);  // 插入后自动排序
    }
    
    long long ans = 0;
    
    while (s.size() > 1) {
        // 取最小的两个
        auto it = s.begin();      // 指向最小元素
        long long a = *it;
        s.erase(it);               // 删除最小元素
        
        it = s.begin();            // 现在指向第二小的
        long long b = *it;
        s.erase(it);               // 删除第二小的
        
        long long sum = a + b;
        ans += sum;
        s.insert(sum);             // 新堆放回，自动排序
    }
    
    cout << ans << endl;
    
    return 0;
}
```

### multiset关键点

- `begin()`：返回最小元素的迭代器 O(1)
- `erase(iterator)`：删除指定位置的元素 O(log n)
- `insert()`：插入元素并保持有序 O(log n)
- 允许重复元素（这是和set的主要区别）

## 五、时间复杂度分析

### 1. 优先队列实现

- 建堆：O(n)
- 每次操作：pop两次 + push一次 = 3×O(log n) = O(log n)
- 总操作次数：n-1次
- **总时间复杂度：O(n log n)**

### 2. multiset实现

- 建集合：n次插入 = O(n log n)
- 每次操作：取最小O(1) + 删除两个O(log n) + 插入一个O(log n) = O(log n)
- 总操作次数：n-1次
- **总时间复杂度：O(n log n)**

### 3. 你的原始方法（反复排序）

cpp

```
while (a.size() > 1) {
    sort(a.begin(), a.end());     // O(m log m)
    // ... 合并操作
}
```

- 第1次排序：n log n
- 第2次排序：(n-1) log (n-1)
- ...
- 总和 ≈ O(n² log n) ≈ 4亿次操作
- n=10000时可能卡时间过，但n更大时肯定超时

### 4. 空间复杂度

- 所有方法：O(n)

## 六、贪心算法实现要点

### 1. 数据结构选择

- **优先队列**：最简洁，推荐
- **multiset**：思路直观，但代码稍复杂
- **手写堆**：不必要，STL足够

### 2. 注意事项

- 用 `long long` 防止中间结果溢出（虽然题目保证结果<2³¹，但中间sum可能更大）
- 关闭同步流加速输入输出
- 循环条件 `size() > 1`，不是 `> 0`

### 3. 错误写法避免

cpp

```
// 错误：每次都重新排序
sort(a.begin(), a.end());
while (a.size() > 1) {
    // 这样只排序一次不对，因为新堆插入后顺序乱了
}

// 错误：直接相邻合并
for (int i = 0; i < n; i += 2) {
    // 这样忽略了新堆可能比后面的小
}
```

## 七、经验总结

### 1. 贪心识别

- 看到"最小代价""最优次序"等问题，考虑贪心
- 合并果子是典型的哈夫曼编码模型

### 2. 数据结构选择原则

- 需要频繁取最小值 → 优先队列
- 需要保持有序且允许重复 → multiset
- 数据量小 → 简单方法也可行，但要考虑扩展性

### 3. 时间复杂度意识

- n=10⁴：O(n²)≈1亿次，可能过但危险
- O(n log n)≈13万次，稳过
- 养成估算复杂度的习惯

### 4. 代码优化技巧

- `ios::sync_with_stdio(false)` 和 `cin.tie(nullptr)` 加速
- 用 `long long` 防溢出
- 循环条件用 `size() > 1` 而不是计数

### 5. 本题收获

- 理解了哈夫曼树的构造过程
- 掌握了优先队列和multiset的使用
- 体会了贪心算法的正确性证明
- 学会了不同数据结构的优劣比较
