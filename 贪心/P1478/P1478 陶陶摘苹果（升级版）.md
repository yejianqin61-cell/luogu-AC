# P1478 陶陶摘苹果（升级版）

## 题目描述

又是一年秋季时，陶陶家的苹果树结了 $n$ 个果子。陶陶又跑去摘苹果，这次他有一个 $a$ 公分的椅子。当他手够不着时，他会站到椅子上再试试。

这次与 NOIp2005 普及组第一题不同的是：陶陶之前搬凳子，力气只剩下 $s$ 了。当然，每次摘苹果时都要用一定的力气。陶陶想知道在 $s<0$ 之前最多能摘到多少个苹果。

现在已知 $n$ 个苹果到达地上的高度 $x_i$，椅子的高度 $a$，陶陶手伸直的最大长度 $b$，陶陶所剩的力气 $s$，陶陶摘一个苹果需要的力气 $y_i$，求陶陶最多能摘到多少个苹果。

## 输入格式

第 $1$ 行：两个数 苹果数 $n$，力气 $s$。

第 $2$ 行：两个数 椅子的高度 $a$，陶陶手伸直的最大长度 $b$。

第 $3$ 行~第 $3+n-1$ 行：每行两个数 苹果高度 $x_i$，摘这个苹果需要的力气 $y_i$。

## 输出格式

只有一个整数，表示陶陶最多能摘到的苹果数。

## 输入输出样例 #1

### 输入 #1

text

```
8 15
20 130
120 3
150 2
110 7
180 1
50 8
200 0
140 3
120 2
```

### 输出 #1

text

```

```

## 说明/提示

对于 $100%$ 的数据，$n\leq 5000$, $a\leq 50$, $b\leq 200$, $s\leq 1000$, $x_i\leq 280$, $y_i\leq 100$。

## 题目思路

这道题是典型的**贪心算法**应用，主要思路如下：

### 1. 问题分析

- 陶陶能摘到苹果的条件：**苹果高度 ≤ 椅子高度 + 手长**（即 $x_i ≤ a + b$）
- 在满足高度条件的苹果中，需要消耗力气 $y_i$ 来摘取
- 总力气为 $s$，要在力气用完前摘尽可能多的苹果

### 2. 解题步骤

#### 第一步：筛选可达苹果

遍历所有苹果，找出高度不超过 $a + b$ 的苹果，这些才是可能摘到的苹果。

#### 第二步：贪心策略

在可摘的苹果中，**优先摘需要力气最小的苹果**。这是因为：

- 目标是数量最大化
- 相同力气下，摘小力气的苹果可以获得更多数量
- 这是典型的"资源有限，收益最大化"问题

#### 第三步：实现方法

1. 将所有可达苹果按所需力气从小到大排序
2. 依次摘取，直到剩余力气不足以摘下一个苹果

### 3. 算法复杂度

- 时间复杂度：$O(n \log n)$，主要来自排序
- 空间复杂度：$O(n)$

## AC代码

cpp

```
#include <bits/stdc++.h>
using namespace std;

// 定义苹果结构体
struct Apple {
    int height;     // 苹果高度
    int strength;   // 摘苹果需要的力气
};

// 按力气从小到大的比较函数
bool compareByStrength(const Apple& a, const Apple& b) {
    return a.strength < b.strength;
}

int main() {
    int n, s, a, b;
    cin >> n >> s;          // 输入苹果数量和剩余力气
    cin >> a >> b;          // 输入椅子高度和手长
    
    int maxHeight = a + b;   // 能够到的最大高度
    vector<Apple> apples(n);
    
    // 读取所有苹果数据
    for(int i = 0; i < n; i++) {
        cin >> apples[i].height >> apples[i].strength;
    }
    
    // 筛选能够到的苹果
    vector<Apple> validApples;
    for(int i = 0; i < n; i++) {
        if(apples[i].height <= maxHeight) {
            validApples.push_back(apples[i]);
        }
    }
    
    // 将能够到的苹果按所需力气从小到大排序
    sort(validApples.begin(), validApples.end(), compareByStrength);
    
    // 贪心：优先摘力气小的苹果
    int usedStrength = 0;    // 已使用的力气
    int count = 0;           // 已摘的苹果数量
    
    for(int i = 0; i < validApples.size(); i++) {
        // 如果摘这个苹果后力气不会用完
        if(usedStrength + validApples[i].strength <= s) {
            usedStrength += validApples[i].strength;
            count++;
        } else {
            // 力气不够了，后面的苹果力气更大，直接结束
            break;
        }
    }
    
    cout << count << endl;
    
    return 0;
}
```

## 贪心算法总结

### 什么是贪心算法？

贪心算法（Greedy Algorithm）是一种在每一步选择中都采取当前状态下最优的选择，从而希望导致结果是全局最优的算法策略。

### 贪心算法的特点

1. **局部最优**：每一步都选择当前最优解
2. **不可回溯**：做出选择后不会改变
3. **高效性**：通常时间复杂度较低

### 贪心算法的适用条件

1. **最优子结构**：问题的最优解包含子问题的最优解
2. **贪心选择性质**：可以通过局部最优选择得到全局最优解

### 本题的贪心应用

在陶陶摘苹果问题中：

- **局部最优**：在当前剩余力气下，选择摘需要力气最小的可达苹果
- **为什么正确**：因为目标是数量最大化，摘小力气的苹果不会影响摘其他苹果的可能性，且能最大化数量

### 常见贪心问题类型

1. **活动安排问题**：选择结束时间最早的活动
2. **区间覆盖问题**：选择覆盖点最多的区间
3. **背包问题（部分背包）**：选择单位价值最高的物品
4. **哈夫曼编码**：合并权值最小的两个节点
5. **最小生成树**：Prim算法、Kruskal算法

### 贪心 vs 动态规划

| 特点       | 贪心算法     | 动态规划   |
| ---------- | ------------ | ---------- |
| 决策方式   | 局部最优     | 全局考虑   |
| 子问题     | 独立         | 重叠       |
| 时间复杂度 | 通常较低     | 可能较高   |
| 适用条件   | 贪心选择性质 | 最优子结构 |

### 使用贪心算法的注意事项

1. 需要证明贪心策略的正确性
2. 注意问题是否具有贪心选择性质
3. 贪心算法不适用于所有优化问题

通过这道题，我们可以看到贪心算法在解决"资源有限，收益最大化"类问题时的强大之处。
