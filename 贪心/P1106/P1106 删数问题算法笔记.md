# P1106 删数问题算法笔记

## 1. 题目核心：贪心算法

本题的目标是删除 $k$ 个数字使剩下的数最小。其核心逻辑在于**位值原理**：高位数字越小，整体数值越小。

- **贪心策略**：每次删除一个数字时，都尽可能让剩下的高位数字变小。
- **寻找“山峰”**：从左往右看，如果某位数字比右边的数字大（$s[i] > s[i+1]$），删掉它后，后面的小数会爬到更高位，这是当前收益最大的选择。

------

## 2. 方案一：找山峰模拟法（逻辑直观）

### 思路

循环 $k$ 次，每轮执行：

1. 遍历当前数字序列，找到第一个满足 `shuzi[i] > shuzi[i+1]` 的位置，将其删除。
2. 如果遍历完都没找到（说明序列是升序的，如 `12345`），就删除最后一个数字。
3. 删完一个后立即跳出内层循环，进入下一轮删除。

### AC 代码（基于你的优化版本）

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    string n;
    int k;
    cin >> n >> k;

    vector<int> shuzi;
    for (char c : n) shuzi.push_back(c - '0');
    
    while (k > 0) {
        bool dele = false;
        int len = shuzi.size();
        for (int i = 0; i < len - 1; i++) {
            if (shuzi[i] > shuzi[i + 1]) { // 发现第一个降序对
                shuzi.erase(shuzi.begin() + i);
                dele = true;
                break;
            }
        }
        if (!dele) shuzi.pop_back(); // 若全升序，删末尾最大的
        k--;
    }
    
    // 边界处理：前导零与全零情况
    int begin = 0;
    while (begin < (int)shuzi.size() - 1 && shuzi[begin] == 0) {
        begin++;
    }
    
    // 输出
    for (int i = begin; i < shuzi.size(); i++) {
        cout << shuzi[i];
    }
    return 0;
}

### 易错与边界处理点

- **越界检查**：在判断 `shuzi[i+1]` 时，`i` 只能到 `size - 2`（即 `i < len - 1`）。
- **全零保底**：`while` 循环中的 `begin < (int)shuzi.size() - 1` 非常关键。它保证了即使全是 `0`，也会留下最后一位 `0` 不被跳过，避免输出为空。
- **类型安全**：`.size()` 返回无符号整数，做减法前强转 `(int)` 可以避免 `0 - 1` 变成巨大正整数的逻辑错误。

## 3. 方案二：单调栈优化法（进阶高效）

### 思路

通过“栈”的思想，只扫描一遍数字即可完成任务。

1. 遍历字符串中的每个数字。
2. **比较与剔除**：如果当前数字比栈顶数字小，且手里还有删除名额（$k>0$），就弹出栈顶，直到栈顶不再比它大或名额用完。
3. **入栈**：将当前数字推入栈中。
4. **收尾**：如果扫描完名额还有剩，从栈末尾（大数）直接切掉。

### AC 代码

#include <bits/stdc++.h>
using namespace std;

int main() {
    string n;
    int k;
    cin >> n >> k;

    string res = ""; // 用 string 当作栈使用
    for (char c : n) {
        // 核心：只要新来的更小，就踢走前面的大数
        while (k > 0 && !res.empty() && res.back() > c) {
            res.pop_back();
            k--;
        }
        res.push_back(c);
    }
    
    // 处理还没用完的 k
    while (k > 0 && !res.empty()) {
        res.pop_back();
        k--;
    }
    
    // 处理前导零
    int begin = 0;
    while (begin < (int)res.size() - 1 && res[begin] == '0') begin++;
    
    // 输出结果，注意为空判断
    if (res.empty()) cout << 0;
    else cout << res.substr(begin);
    
    return 0;
}

## 4. 总结对比

| **特性**       | **模拟法**                    | **单调栈法**             |
| -------------- | ----------------------------- | ------------------------ |
| **时间复杂度** | $O(n \cdot k)$ (约为 $250^2$) | $O(n)$ (线性扫描)        |
| **代码难度**   | 逻辑直观，但删除操作多        | 逻辑稍抽象，但代码简洁   |
| **适用范围**   | 适合位数较少的简单题目        | 适合位数上万的高阶算法题 |
| **核心技巧**   | 寻找第一个 `a[i] > a[i+1]`    | 维护一个递增的序列       |
