# 火柴棒等式（NOIP 2008 提高组）解题总结

## 📝 题目回顾

**题目**：用 $n$ 根火柴棍，可以拼出多少个形如 $A+B=C$ 的等式？

**关键规则**：

- 数字 $0-9$ 各有固定的火柴棍数量
- 加号和等号各需要 **2根** 火柴（共 **4根**）
- $A,B,C \geq 0$，且非零数最高位不能是0
- $A \neq B$ 时，$A+B=C$ 与 $B+A=C$ 视为不同等式
- 所有 $n$ 根火柴必须用完

**数据范围**：$1 \leq n \leq 24$

------

## 💡 解题思路

### 1. 核心转化

将问题转化为：

text

```
火柴总数 = A的火柴数 + B的火柴数 + C的火柴数 + 4
```

其中 $C = A + B$

### 2. 确定枚举范围

这是本题最关键也最容易出错的地方！

**错误做法**：凭感觉设定小范围（如 $0-23$）
**正确思路**：根据 $n \leq 24$ 推算最大可能的数字

- 用于数字的火柴最多 $24-4=20$ 根
- 数字1只需要2根火柴，20根最多可以拼出10个1，即数字 $1111111111$（十位数）
- 但实际测试发现，$n=18$ 时已经出现了 $11$（两位数）
- **经验范围**：枚举到 $1000$ 或 $2000$ 足够覆盖所有可能

### 3. 预处理优化

预先计算 $0-2000$ 每个数字所需的火柴数，避免重复计算。

------

## 🚫 常见错误及解决方案

### 错误1：循环范围太小 ❌

cpp

```
for(i=0;i<=23;i++)  // 错误！只能找到0-23
```

**后果**：漏掉两位数及以上的数字，导致答案偏小

**解决**：根据题目数据范围合理扩大，一般到 $1000$ 足够

### 错误2：循环变量被意外修改 ❌

cpp

```
while(i>9) {
    temp1=i%10;
    i/=10;  // 错误！修改了循环变量 i
}
```

**后果**：循环失控，程序无法正常运行

**解决**：使用临时变量或封装成函数

### 错误3：未考虑加号和等号的4根火柴 ❌

cpp

```
if(cnt_i+cnt_j+cnt_p == n)  // 错误！忘记加4
```

**后果**：永远找不到符合条件的等式

**解决**：`cnt_i + cnt_j + cnt_p + 4 == n`

### 错误4：未处理数字0的情况 ❌

cpp

```
while(num>0) {  // 当num=0时，循环不执行
    cnt += hc[num%10];
    num /= 10;
}
```

**后果**：数字0的火柴数被计算为0（实际应为6）

**解决**：单独处理 `num == 0` 的情况

## ✅ 正确代码模板

cpp

```
#include <bits/stdc++.h>
using namespace std;

int hc[] = {6, 2, 5, 5, 4, 5, 6, 3, 7, 6};
int cost[2001];

// 计算数字x所需的火柴数
int count(int x) {
    if (x == 0) return hc[0];
    
    int cnt = 0;
    while (x > 0) {
        cnt += hc[x % 10];
        x /= 10;
    }
    return cnt;
}

int main() {
    int n;
    cin >> n;
    
    // 预处理0-2000的火柴数
    for (int i = 0; i <= 2000; i++) {
        cost[i] = count(i);
    }
    
    int ans = 0;
    // 枚举A和B
    for (int a = 0; a <= 1000; a++) {
        for (int b = 0; b <= 1000; b++) {
            int c = a + b;
            if (cost[a] + cost[b] + cost[c] + 4 == n) {
                ans++;
            }
        }
    }
    
    cout << ans;
    return 0;
}
```

## 🎯 经验总结

### 1. **范围估算要合理**

- 不要凭感觉设范围
- 根据题目给出的最大值反推可能的最大数字
- 不确定时可以适当放大（但注意性能）

### 2. **避免修改循环变量**

- 循环变量是循环的"生命线"，千万不要在循环体内修改
- 需要修改时，用临时变量替代

### 3. **封装函数提高代码质量**

- 将重复代码提取成函数
- 提高代码可读性，减少出错概率
- 便于调试和修改

### 4. **边界条件要周全**

- 数字0的特殊处理
- 加号和等号的4根火柴不要忘
- 非零数最高位不能是0（题目隐含，但本题枚举范围已自动满足）

### 5. **预处理优化**

- 预先计算好每个数字的火柴数
- 避免三重循环内的重复计算
- 空间换时间的典型应用

------

## 🔍 扩展思考

如果题目改成 $A \times B = C$ 呢？

- 范围可能需要调整（乘法结果更大）
- 基本思路相同，但枚举上限要重新估算

如果 $n$ 更大（如 $n \leq 100$）呢？

- 需要更高效的算法（如DP）
- 简单的枚举会超时
