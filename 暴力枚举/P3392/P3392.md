# WA到AC：荷兰国旗问题解题思路复盘

## 问题背景

这是一个将矩形分成白色、蓝色、红色三个水平条纹的问题，要求：

- 至少一行白色（顶部）
- 至少一行蓝色（中间）
- 至少一行红色（底部）
- 每行必须是单一颜色
- 求最少需要涂改的格子数

## 代码演进过程

### V1：最直观但错误的解法

cpp

```
// 错误代码片段
for(i=0;i<m;i++){
    if(matrix[0][i]!='W')cnt++;  // 第1行必须白
}
for(j=0;j<m;j++){
    if(matrix[1][j]!='B')cnt++;  // 第2行必须蓝
}
for(i=2;i<n;i++){
    for(j=0;j<m;j++){
        if(matrix[i][j]!='R')cnt++;  // 其余行必须红
    }
}
```

**思维误区**：

- ❌ 固定思维：认为第1行必须是白、第2行必须是蓝
- ❌ 忽略了题目的灵活性：白色可以是1行，也可以是2行、3行...

### V2：尝试枚举但索引混乱

cpp

```
for(i=1;i<=n-2;i++) {
    for(j=i+1;j<=n-1;j++) {
        // 白色区域：0到i-1
        // 蓝色区域：i+1到j-1  ← 错误！少了第i行
        // 红色区域：j到n-1
    }
}
```

**思维误区**：

- ❌ 边界处理错误：蓝色区域起始行应该是`i`，不是`i+1`
- ❌ 混淆了"行数"和"行索引"的概念

### V3：变量重置位置错误

cpp

```
for(w=1;w<=n-2;w++) {
    for(b=w+1;b<=n-1;b++) {
        // 计算cnt...
    }
    mi = min(cnt, mi);  // 更新位置错误！
}
```

**思维误区**：

- ❌ 循环嵌套的逻辑混乱：最小值更新放在了错误的位置
- ❌ 没有理解每个划分都应该独立计算并比较

### V4：最终AC版本

cpp

```
for(int w=1; w<=n-2; w++) {              // 白色行数
    for(int b=w+1; b<=n-1; b++) {        // 蓝色结束行
        int cnt = 0;
        
        // 白色区域：[0, w-1]
        for(int k=0; k<w; k++) {
            for(int p=0; p<m; p++) {
                if(matrix[k][p] != 'W') cnt++;
            }
        }
        
        // 蓝色区域：[w, b-1]  ← 关键修正
        for(int k=w; k<b; k++) {
            for(int p=0; p<m; p++) {
                if(matrix[k][p] != 'B') cnt++;
            }
        }
        
        // 红色区域：[b, n-1]
        for(int k=b; k<n; k++) {
            for(int p=0; p<m; p++) {
                if(matrix[k][p] != 'R') cnt++;
            }
        }
        
        mi = min(mi, cnt);  // 每个划分都更新
    }
}
```

## 经验总结

### 1. **读懂题目约束是关键**

- 题目说"至少一行"，意味着变量可以取1到n-2
- 不是固定分配，而是需要枚举所有可能性

### 2. **索引与数量的区分**

- 行数：白色有w行 → 行索引范围 [0, w-1]
- 蓝色有b行？不对，b应该是边界索引，不是行数
- 最终采用：白色[0,w-1]，蓝色[w,b-1]，红色[b,n-1]

### 3. **循环嵌套的逻辑要清晰**

- 外层循环枚举所有可能的划分
- 内层对每种划分独立计算
- 及时更新最小值

### 4. **调试技巧**

- 遇到WA时，先用小数据手动模拟
- 检查边界条件（w=1, b=2 时是否正确？）
- 验证每个循环变量的取值范围

### 5. **代码可读性很重要**

- 变量命名要有意义（w代表white，b代表blue boundary）
- 注释清晰说明每个循环的作用
- 这样容易发现逻辑错误

## 核心收获

**枚举类问题的通用思路**：

1. 确定所有可能的变量取值范围
2. 对每种组合独立计算
3. 记录最优解
4. 特别注意边界条件和索引转换

这道题教会我：**看似简单的问题往往隐藏在细节中，严谨的边界处理比华丽的算法更重要**。
