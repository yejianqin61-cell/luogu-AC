## 题目描述

可是……这个篮球场，好像很久没有使用过的样子啊……

里面堆满了学校的各种杂物呢……

我们 Aqours 的成员要怎么在里面列队站下呢？


我们浦之星女子学院的篮球场是一个 $R$ 行 $C$ 列的矩阵，其中堆满了各种学校的杂物 (用 `#` 表示)，空地 (用 `.` 表示) 好像并不多的样子呢……

我们 Aqours 现在已经一共有 $K$ 个队员了，要歌唱舞蹈起来的话，我们得排成一条 $1\times K$ 的直线，一个接一个地站在篮球场的空地上呢 (横竖均可)。

我们想知道一共有多少种可行的站位方式呢。

Aqours 的真正的粉丝的你，能帮我们算算吗？

## 输入格式

第一行三个整数 $R, C, K$。

接下来的 $R$ 行 $C$ 列，表示浦之星女子学院篮球场。

## 输出格式

总共的站位方式数量。

## 输入输出样例 #1

### 输入 #1

```
5 5 2
.###.
##.#.
..#..
#..#.
#.###

```

### 输出 #1

```
8
```

## 说明/提示

|           |    $R$     |    $C$     |       $K$        |   备注   |
| :-------: | :--------: | :--------: | :--------------: | :------: |
| $1\sim2$  | $\leq 10$  | $\leq 10$  | $\leq \min(R,C)$ |    无    |
| $3\sim4$  | $\leq 100$ | $\leq 100$ |     $\leq 1$     |    无    |
| $5\sim6$  | $\leq 100$ | $\leq 100$ | $\leq \min(R,C)$ | 没有障碍 |
| $7\sim10$ | $\leq 100$ | $\leq 100$ | $\leq \min(R,C)$ |    无    |

对于所有数据，$1 \leq R,C \leq 100$，$1 \leq k \leq \min(R,C)$。

# 从40分到100分：一次编程竞赛的经验总结

## 📝 问题回顾

在做洛谷P3654「First Step」这道题时，我的代码从40分到100分的转变，让我深刻认识到了编程竞赛中的一个重要道理。

## ❌ 错误代码（40分版本）

cpp

```
if(cnt >= k)
{
    cnt = 1;  // 错误：重置计数器
    ans++;
}
```

## ✅ 正确代码（100分版本）

cpp

```
if(cnt >= k)
{
    ans++;    // 正确：只计数，不重置
    // cnt保持原值，继续增长
}
```

## 📊 差距有多大？

用具体数据对比：

**输入**：`1行6列，K=3，全部是空地(......)`

| 位置     | 正确逻辑      | 错误逻辑             |
| -------- | ------------- | -------------------- |
| j=0      | cnt=1         | cnt=1                |
| j=1      | cnt=2         | cnt=2                |
| j=2      | cnt=3 → ans=1 | cnt=3 → ans=1, cnt=1 |
| j=3      | cnt=4 → ans=2 | cnt=2                |
| j=4      | cnt=5 → ans=3 | cnt=3 → ans=2, cnt=1 |
| j=5      | cnt=6 → ans=4 | cnt=2                |
| **最终** | **ans=4**     | **ans=2**            |

**整整少算了2种站位！** 😱

## 💡 为什么会犯这个错误？

1. **过度依赖题目样例**：只根据题目给的样例测试，没有考虑更多情况
2. **思维定式**：想当然地认为找到一组后要重置计数器
3. **缺乏自测**：没有自己设计更多的测试用例

## 🎯 经验教训

### 1. 不要只依赖题目样例

- 样例只是帮你理解题意
- 样例往往是最简单的情况
- 样例通过不代表代码正确

### 2. 自己设计测试数据

text

```
// 测试长连续空位
1 10 3
..........

// 测试K=2的情况
3 3 2
...
...
...

// 测试混合障碍
3 5 2
.#...
..#..
...$.

// 测试边界值
3 3 3  // K取最大值
...
...
...

// 测试极端情况
3 3 2
###  // 全是障碍
###
###
```

### 3. 多考虑特殊情况

- 最大K值
- 最小K值(K=1)
- 全空场地
- 全障碍场地
- 长连续序列
- 交替出现的障碍

## 🌟 核心收获

**90%的逻辑正确 + 10%的细节错误 = 40分**
**100分的代码 = 正确逻辑 + 完美的细节处理**

在编程竞赛中，**细节决定成败**！一个看似微小的判断，可能就让你的分数从100掉到40。

## 📌 以后做题 checklist

- 理解题目意思
- 通过样例测试
- 自己设计5-10组测试数据
- 考虑边界情况
- 用手动模拟验证小数据
- 检查每个细节逻辑
