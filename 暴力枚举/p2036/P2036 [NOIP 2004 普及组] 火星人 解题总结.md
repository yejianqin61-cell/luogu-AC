## 题目原文

**题目描述**
人类终于登上了火星的土地并且见到了神秘的火星人。人类和火星人都无法理解对方的语言，但是我们的科学家发明了一种用数字交流的方法。火星人用一种非常简单的方式来表示数字——掰手指。火星人只有一只手，但这只手上有成千上万的手指，这些手指排成一列，分别编号为 1,2,3,...。火星人的任意两根手指都能随意交换位置，他们就是通过这方法计数的。

一个火星人用一个人类的手演示了如何用手指计数。当手指按正常顺序排列时，形成了5位数12345；交换无名指和小指的位置时，形成12354；把五个手指的顺序完全颠倒时，形成54321。在所有能够形成的120个5位数中，12345最小，它表示1；12354第二小，它表示2；54321最大，它表示120。

现在你有幸成为了第一个和火星人交流的地球人。一个火星人会让你看他的手指，科学家会告诉你要加上去的很小的数。你的任务是，把火星人用手指表示的数与科学家告诉你的数相加，并根据相加的结果改变火星人手指的排列顺序。

**输入格式**

- 第一行一个正整数 N，表示火星人手指的数目（1 ≤ N ≤ 10000）
- 第二行是一个正整数 M，表示要加上去的小整数（1 ≤ M ≤ 100）
- 下一行是 1 到 N 这 N 个整数的一个排列，用空格隔开，表示火星人手指的排列顺序

**输出格式**
N 个整数，表示改变后的火星人手指的排列顺序，每两个相邻的数中间用一个空格分开

**输入输出样例**

text

```
输入：
5
3
1 2 3 4 5

输出：
1 2 4 5 3
```

**数据范围**

- 对于 30% 的数据，N ≤ 15
- 对于 60% 的数据，N ≤ 50
- 对于 100% 的数据，N ≤ 10000

## 题目分析

### 核心问题

给定一个 1~N 的排列，求这个排列后面的第 M 个字典序排列。

### 关键点

1. **数据范围**：N 最大 10000，M 最大 100
2. **时间复杂度**：不能生成所有排列（N! 太大）
3. **空间复杂度**：不能存储所有排列
4. **结果保证**：题目保证结果在手指能表示的范围内

### 初步思路误区

一开始想用 DFS 生成所有排列，然后找第 M 个。但 N=10000 时，排列数是天文数字，根本不可能。

------

## 解题思路

### 正确思路

**步进法**：从当前排列开始，重复 M 次"找下一个字典序排列"。

### 下一个字典序排列算法（核心）

#### 算法步骤

1. **从右向左找第一个上升位置**
   - 找到第一个 `a[i] < a[i+1]` 的位置 i
   - 如果找不到，说明已经是最大排列
2. **从右向左找第一个比 a[i] 大的数**
   - 在 i 的右边部分，从右向左找第一个大于 a[i] 的数 a[j]
   - 因为右边部分是降序的，所以找到的就是刚好比 a[i] 大的数
3. **交换 a[i] 和 a[j]**
   - 此时 i 位置变成了"刚好比原来大"的数
4. **反转 i+1 到末尾**
   - 将 i+1 到末尾的部分反转（原来是降序，反转后变成升序，即最小排列）

#### 举例说明

排列：`1 5 8 4 7 6 5 3 2`

1. 从右找第一个上升：4 < 7，所以 i=3（指向4）
2. 从右找第一个大于4的数：找到5（j=6）
3. 交换：`1 5 8 5 7 6 4 3 2`
4. 反转 i+1 到末尾：`7 6 4 3 2` 反转成 `2 3 4 6 7`
5. 最终结果：`1 5 8 5 2 3 4 6 7`

**问题**：参数类型错误，导致数组访问异常

cpp

```
for(i=n-1;i>=0;i--){
    if(finger[i]<finger[i+1])  // 当 i=n-1 时越界

```

**问题**：应该从 n-2 开始遍历

### 版本3：缺少break

cpp

```
for(i=n-2;i>=0;i--){
    if(finger[i]<finger[i+1]){

    }

```

**问题**：找到后没有及时跳出，导致找到的不是最右边的上升位置

只交换了位置，没有把后面的部分反转成升序，导致结果错误

**问题**：这里是对的，但和版本6的问题混淆了

## 最终AC代码

cpp

```
#include <bits/stdc++.h>
using namespace std;

int n, m;

void next_permutation(int* finger) {
    // 1. 从右向左找第一个上升的位置
    int i = n - 2;
    while (i >= 0 && finger[i] >= finger[i + 1]) {
        i--;
    }
    
    // 2. 如果不是最后一个排列
    if (i >= 0) {
        // 3. 从右向左找第一个大于 finger[i] 的数
        int j = n - 1;
        while (j >= 0 && finger[j] <= finger[i]) {
            j--;
        }
        // 4. 交换
        swap(finger[i], finger[j]);
    }
    
    // 5. 反转 i+1 到末尾的部分
    reverse(finger + i + 1, finger + n);
}

int main() {
    cin >> n >> m;
    int finger[n];
    for (int i = 0; i < n; i++) {
        cin >> finger[i];
    }
    
    // 执行 M 次下一个排列
    for (int i = 0; i < m; i++) {
        next_permutation(finger);
    }
    
    // 输出结果
    for (int i = 0; i < n; i++) {
        if (i > 0) cout << " ";
        cout << finger[i];
    }
    cout << endl;
    
    return 0;
}
```

## 要点总结

### 算法要点

1. **字典序排列**：理解什么是字典序，如何比较两个排列的大小
2. **下一个排列算法**：掌握"找上升→找更大→交换→反转"的四步法
3. **时间复杂度**：O(M·N) 在本题数据范围内可行
4. **边界条件**：处理已经是最大排列的情况

### 代码要点

1. **数组遍历**：从右向左遍历时注意起始索引（n-2 而不是 n-1）
2. **找到后及时break**：避免找到错误的位置
3. **反转操作**：这是最容易遗漏的一步
4. **输出格式**：注意空格的处理，不要有多余空格

### 常见错误

1. **数组越界**：`finger[i+1]` 当 i=n-1 时越界
2. **作用域错误**：循环内定义变量导致每次重置
3. **遗漏反转**：只交换不反转，结果不对
4. **索引混淆**：反转部分的起始和结束位置搞错
